# The Artisan Kiln

**The Artisan Kiln** — одностраничное веб-приложение в стиле мастерской керамики: форма заказа ручной керамической плитки с корзиной, визуальным конструктором узора и оформлением заказа. Интерфейс оформлен как печатная бланк-форма с тёплой палитрой, декоративными иллюстрациями и чёткой типографикой.

> Это **фронтенд-демо**: данные хранятся в браузере (Redux), оплата и отправка заказа не подключены к серверу — после успешной валидации формы показывается уведомление.

---

## Для чего этот проект

Приложение имитирует сценарий заказа плитки у мастерской **The Artisan Kiln**:

1. **Выбор коллекции** — клиент указывает количество плиток по каждой позиции (Ocean Wave, Forest Fern, Terracotta Dot, Yellow Star) и видит подытог, доставку и итог.
2. **Визуализация узора** — на сетке 6×6 можно собрать будущий рисунок, перетаскивая мотивы из палитры (drag-and-drop).
3. **Оформление заказа** — контактные данные, адрес, заметки к проекту и способ оплаты с проверкой полей на стороне клиента.

Проект подходит как:

- **портфолио** или учебный пример современного React/Next.js UI;
- **макет** для будущего интернет-магазина плитки или кастомных заказов;
- **полигон** для отработки форм, состояния, адаптивной вёрстки и drag-and-drop.

## Основные возможности

| Блок | Описание |
|------|----------|
| **Корзина** | Таблица позиций: коллекция, превью, количество (кв. футы), цена за единицу, действия. Бесплатная доставка при сумме заказа **> $500**, иначе **$25**. |
| **Конструктор узора** | Сетка 6×6, палитра из 12 мотивов (floral, угловые элементы, волна, папоротник и др.). Перетаскивание с анимацией (Framer Motion). |
| **Оформление заказа** | Имя, email, телефон, адрес, заметки к проекту. Способы оплаты: карта, PayPal, Apple Pay, банковский перевод. Для карты — отдельная валидация (16 цифр, MM/YY, CVV). |
| **Адаптив** | **Десктоп:** три колонки — корзина, конструктор, форма. **Мобильный:** одна прокручиваемая колонка (форма → корзина → оплата). |
| **Шапка и подвал** | Счётчик товаров в корзине, навигация-заглушки (SHOP, COLLECTIONS, ABOUT US и т.д.), декоративные SVG-элементы. |

## Как устроена логика

### Корзина (`cartSlice`)

- Начальный набор — четыре коллекции из `src/data/tiles.ts` с предзаданным количеством.
- Действия: изменить количество, удалить позицию, добавить случайную плитку (кнопка в UI).
- Селекторы: `selectSubtotal`, `selectShipping`, `selectGrandTotal`.

### Дизайн (`designSlice`)

- Сетка `6×6`, ячейка — мотив (`DesignPattern`) или `null`.
- Верхний ряд частично заполнен угловыми элементами по умолчанию.
- Действия: выбрать мотив, поставить на ячейку, удалить, сбросить сетку.

### Оформление заказа

- Схема `checkoutSchema` в `src/lib/validations.ts`.
- При способе оплаты `credit-card` дополнительно проверяются номер карты, срок и CVV.
- `onSubmit` сейчас вызывает `alert` — точка подключения API или платёжного шлюза.

### Адаптивная вёрстка

- Breakpoint `md` (768px): переключение между мобильным и десктопным макетом.
- На больших экранах (`lg`) — декоративные «листья» по бокам страницы.

## Ограничения и заметки

- **Нет бэкенда** — заказы никуда не отправляются, платежи не обрабатываются.
- **Нет авторизации** — кнопки «Log In» и имя пользователя в шапке декоративные.
- **Ссылки навигации** (`SHOP`, `FAQ` и т.д.) ведут на `#` — заглушки.
- **Состояние Redux** сбрасывается при перезагрузке страницы (нет persist).
- Проект использует **Next.js 16** с отличиями от классической документации — при доработке смотрите актуальные гайды в `node_modules/next/dist/docs/`.

## Переменные окружения

Сейчас приложение **не требует** `.env` для локального запуска. При подключении API, аналитики или платежей добавьте, например:

```env
# .env.local (не коммитить — см. .gitignore)
NEXT_PUBLIC_API_URL=https://api.example.com
```

Шаблон можно оформить как `.env.example` и закоммитить без секретов.
